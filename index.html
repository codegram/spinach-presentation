Why?

Because we realized what we loved was...

Gherkin

  • Natural-ish DSL
  • The language behind Cucumber
  • Abstract feature description
  • Focus on business value (In order to...)

What sucks about Cucumber

Step MADNESS

Step ambiguity

Regex-based step matching

Reusable Cucumber steps == Ugly steps

(regexes matching a shitload of different cases)

We love Gherkin, not Cucumber

so ...

/spɪnɪtʃ/

  • Features are just Ruby classes
  • Leverages Gherkin parser
  • Steps are just Ruby methods
  • Supports MiniTest and RSpec, with Capybara, etc

Fighting step madness and ambiguity

  • Each feature has its own steps (no global steps)
  • Explicit reusability through Ruby mixins

Simple architecture

  • Small codebase
  • Fully documented
  • Simple hooks system

Ruby compatibility

  • MRI 1.9
  • Rubinius/JRuby on the works

Web framework compatibility

  • spinach-rails
  • Any rack-based framework (Sinatra, Padrino, etc...)
